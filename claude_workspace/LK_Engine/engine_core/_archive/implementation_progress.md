### 6. 引擎分析系统重构（2026-04-11）

#### 评估器 v3 (`engine/evaluator.py`)
- 修复 `_evaluate_side` 双重计算 bug（旧版对手 debuff 在双方评估中各计一次）
- 修复 `bench_alive` 作用域引用 bug（使用 `in dir()` 的不可靠判断）
- 修复心数估值方向（v2 方向反了：最后一心应最宝贵，现改为边际递增）
- 修复速度评估不对称（对称框架下先手方+150，不再扣后手分）
- HP 评估区分在场精灵（权重 2000）与后备精灵（权重 800/只）
- 属性克制基于实际技能攻击面
- 状态效果惩罚引入 urgency 系数
- 新增技能冷却惩罚（冷却中技能 -40/个）
- 新增天气影响评估（雨天水系+80、沙暴地系+60 等）
- 新增后备精灵对当前敌方的克制覆盖评估
- **新增 `evaluation_to_win_rate()` sigmoid 胜率映射**

#### 搜索引擎 v5 (`engine/search_engine.py`) — 深度优化版
- 同时博弈 Maximin + Expectimax 混合搜索框架
- **迭代加深 + PV-move 优先**：上一层最佳行动在下一层第一个搜索，收紧 alpha 窗口提升剪枝
- **自适应宽度**：叶子层(d=1)玩家5/对手4，中间层6/5，根层8/6 → 节点数减半
- **Futility Pruning**：depth=1 时静态评估 ± 600 偏离 alpha/beta 窗口则直接跳过
- **根节点浅层预排序**：depth≥3 时用 depth=1 快速评估裁剪弱行动（保留前10个）
- 性能参考（1s 迭代加深）：1v1→d6, 2v2→d4, 3v3→d3, **6v6→d3** ← 核心改善
- 对比 v4：6v6 depth=3 从 1.26s/297节点 → **0.37s/166节点**（3.4x 加速）

#### 战斗引擎确定性修复 (`engine/extended_battle_engine.py`)
- 移除 `_determine_order` 中的 `random.random()`，改为基于精灵 ID 哈希的确定性打破平局
- 重写 `_force_end_battle`：不再掷硬币，改为按心数→总HP→存活数逐级比较确定性判定
- 搜索树中不再有任何非确定性因素

#### 胜率分析 (`game_analysis_engine.py` + `api.py`)
- `analyze_state()` 新增 `win_rate` 和 `action_scores`（每个行动的分数+胜率）
- 支持传入 `opponent_action_weights` 和 `confidence` 参数（暗箱模式）
- `/api/analyze` 返回值新增 `win_rate` 和 `action_scores`
- `/api/eval` 返回值新增 `win_rate`
- `/api/simulate` 完全重写：
  - 修复对手视角翻转不完整的 bug（心数/印记/行动索引错误）
  - 改为正确的对手 minimax 搜索（`_find_best_opponent_action`）
  - 新增 `num_games` 参数支持多局模拟统计胜率

### 7. 暗箱对战不确定性系统（2026-04-11）

#### 对手行为预测器 (`agent_runtime/engine/behavior_predictor.py`)
新增模块，三层预测模型：
- **先验层**：博弈常识（攻击默认最高分、换宠基础分、聚能基础分）
- **局面层**：根据 HP/能量/属性克制动态调整
  - 残血 → 换宠权重×3
  - 低能量 → 聚能权重×2.5
  - 被克制 → 换宠权重×2
  - 克制对手 → 攻击权重×1.8
- **历史层**：基于对手已观察行为模式修正（需 ≥3 条记录）
  - 攻击倾向、换宠倾向、低血换宠率、克制攻击率
  - 技能偏好、换宠目标偏好
- **预读分析** (`analyze_reads`)：
  - 读切（预判对手换宠）、读攻（预判对手攻击）、读能（预判对手聚能）
  - 输出建议反制策略和预读失败风险

#### 推荐服务升级 (`agent_runtime/engine/recommendation_service.py`)
- 集成 `BehaviorPredictor`，每次推荐时自动进行对手行为预测
- 推荐结果新增 `opponent_prediction`（对手行动概率分布）和 `read_opportunities`（预读机会）
- 搜索评分支持使用预测概率做加权期望（通过 `_score_fixed_action` 传入权重）
