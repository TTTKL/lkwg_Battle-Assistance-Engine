## 已实现（概览）

### 引擎主干
- `BattleState`/`PetInstance` 为状态真源，搜索依赖 `copy()` 深拷贝
- 完整回合推进：换宠、聚能、首领化、愿力冲击、同时行动、先手判定、伤害结算、回合末结算、心数扣除
- 战斗引擎主流程已拆出准备阶段、技能阶段、伤害段、回合结束等辅助模块
- REST API / CLI / 搜索 / 评估主链路可运行

### 基础机制
- 状态：中毒、中毒印记、灼烧、冻结、寄生、眩晕
- 印记：星陨、蓄电、蓄势、攻击、风起、龙噬、湿润、迟缓、光合、降灵、棘刺、凝霜
- 技能机制：应对、迅捷、迸发、蓄力、折返、复活、驱散、吸血、多段攻击、使用次数追加、技能永久成长、槽位/传动主干规则
- 属性倍率已统一收口到引擎：多属性按连乘结算，重复属性去重，双重克制 4x 压 3x，双重抵抗保留 0.25x
- 奉献系统主干
- 首领化/愿力冲击互斥 + 血脉约束

### 特性系统
155 种独立特性，覆盖情况：~136 完整实现、9 近似实现、2 无法实现（缺数据）、剩余为未注册空特性。

### 搜索与评估
- Minimax + Alpha-Beta（同时博弈 Maximin + Expectimax 混合）
- 迭代加深 + PV-move + 自适应宽度 + Futility Pruning
- 评估器 v3：心数差、HP比例、存活数量、能量、克制、Buff/Debuff、状态、印记、队伍战力、首领化/愿力冲击
- 胜率 sigmoid 映射 + 全行动评分

### 已实现的复杂技能（部分列举）
感电、落雷、叠势、能量刃、流星火雨、齿轮扭矩、绵里藏针、孢子爆散、借用、隐藏条款、伪造账单、恶念交换、欺诈契约、炎爆术、逆向演化、过载回路、均势、荟萃、雷暴、镜像反射、疾风连袭、取念、复写、贮藏、富养化、击鼓传花、甜心续航、超新星馈赠、气沉丹田、激怒、操控、精神扰乱、聒噪、摇篮曲 等。

## 近似实现（9 种特性）

| 特性 | 偏差 |
|------|------|
| 不朽 | 缺少3回合延迟，死亡时立即以1HP复活 |
| 先知 | 无条件判断，入场时固定给予速度+5层、双攻+5层 |
| 预警 | 无条件判断，入场时固定给予速度+5层 |
| 哨兵 | 无条件判断，每次使用技能后脱离 |
| 威慑 | 依赖 `_just_countered` 标记，时序可能不准确 |
| 石头大餐 | 实现时机正确，仅在引擎层处理 |
| 吟游之弦 | 已注册但无效果，需重构支持多印记共存 |
| 绵里藏针 | 按"上回合未受到任何伤害"判定，持续伤害也计入 |
| 借用/取念/复写 | 使用确定性等价随机，不展开真实随机分支 |

## 当前边界

1. 部分槽位/传动衍生的非伤害效果仍未完整
2. 奉献系统与全部特性的完整联动仍有残缺
3. 网页端默认已切到引擎权威状态；仍需继续清理少量静态数据显示/读取残留
4. 少量需要额外外部数据的特性/规则
5. 策略层已新增 `engine/strategy_mechanics.py` 作为印记/特性近似与占位统一入口；未完善机制应优先挂到该框架，而不是继续散落到 `evaluator.py`

---

## 策略层框架

- 入口：`engine/strategy_mechanics.py`
- 用途：承接“机制已进引擎，但策略估值未细化”以及“机制未完全实现，但策略不能完全忽略”的印记/特性
- 当前 `Evaluator` 已通过该模块统一评估：
  - 场上印记
  - 未完善特性的保守占位分
- 详细说明见：`docs/strategy_framework.md`
