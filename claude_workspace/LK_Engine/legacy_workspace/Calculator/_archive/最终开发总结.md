# 洛克王国对战引擎 - 当前状态总结

更新日期：2026-04-10

## 当前定位

当前网页已经从“多功能展示页”收敛为“以引擎为主体的对战工作台”。

产品主线已经明确：

1. 首页默认服务暗箱分析，而不是明箱全知计算。
2. 用户重点配置我方队伍与技能。
3. 敌方通过阵容导入 + 战斗中的观测事件逐步收紧搜索空间。
4. 引擎输出不仅给行动建议和优劣评分，也给一个可调整的预估胜率。

## 已完成工作

### 1. 首页与布局重构

- 首页默认进入战斗页，图鉴/模型类内容不再占据主路径。
- 主界面视觉改成更成熟、简约、偏工程工具的风格。
- 战斗页形成“主战区 + 右侧引擎摘要”的工作台布局。
- 冗余分析块已从日志区移除，日志退回辅助位置。

### 2. 引擎模式拆分

- 引擎已拆成两个独立入口：
  - `HiddenBoxEngine`
  - `ClearBoxEngine`
- 两者复用底层搜索逻辑，但分析状态构造已分离。
- 当前默认模式为暗箱。
- 明箱保留，但已后置，不再作为首页核心流程。

### 3. 暗箱模式主流程落地

- 暗箱模式下不再要求手工配置敌方精确技能。
- 若敌方未配置技能，会从该精灵技能池中自动挑选候选技能。
- 敌方技能列表已转为“观测输入面板”：
  - 已观测技能会被标记。
  - 未确认技能会显示为预测。
  - 点击敌方技能可登记为已观测。

### 4. 引擎输出增强

- 右侧“引擎输出”会显示：
  - 当前推荐动作
  - 优劣评分
  - 预估胜率
  - 搜索深度
  - 搜索节点数
  - 剪枝数
  - 耗时
- 暗箱模式下，胜率已做保守化处理，避免在不完整信息下过于极端。

### 5. 暗箱敌方参数建模

敌方不再按单一固定个体假设处理，而是按概率与区间建模。

- 默认保留“敌方个体值可能低于 8”的可能。
- 当前基础设定是：
  - 主分布：8-10
  - 少量低个体保留：0-7
- 搜索使用的是动态调整后的期望个体，而不是写死的单点值。
- 敌方面板会显示估计值和区间，而不是伪装成精确真值。

### 6. 暗箱证据收紧链路

当前暗箱模型已经支持以下证据逐步收紧：

- 技能观测
  - 记录敌方已观测技能。
  - 减少未观测技能的搜索权重。

- 速度线
  - 根据同优先级先后手结果，收紧敌方速度个体区间。

- 攻击线
  - 根据敌方对我方造成的实际伤害，收紧敌方物攻/魔攻个体区间。

- 防御线
  - 根据我方对敌方造成的实际伤害，收紧敌方物防/魔防个体区间。

### 7. 按宠物独立保留暗箱证据

- 每只导入到战斗中的宠物现在都有独立观测键。
- 换宠后，技能/速度/攻防约束不会串到别的宠物上。
- 同种精灵多只并存时，也能分开保留各自证据。
- 当前摘要中已增加“当前敌方槽位”提示。

## 当前网页中用户能看到的暗箱反馈

- 当前敌方已观测技能数
- 已观测技能名称
- 敌方低个体保留比例
- 当前敌方槽位
- 参数线索
  - 例如速度个体推断
  - 物攻/魔攻个体推断
  - 物防/魔防个体推断

## 当前实现边界

- 现在的胜率是基于评分映射的工程化近似值，不是严格统计胜率。
- 伤害反推与速度反推目前是粗区间推断，强调稳健，不追求一步拟合到精确个体。
- 暗箱的证据收紧已可用，但还不是完整概率图模型。
- 明箱模式仍可用，但当前不再是首页主要使用路径。

## 下一步可以考虑的方向

### 优先级高

1. 做“当前敌方证据面板”
   - 单独展示当前敌方这只宠已收集到的技能、速度、攻防线索。
   - 让暗箱推断更可解释。

2. 做“按宠物的历史证据视图”
   - 不只看当前出战宠。
   - 能查看敌方队伍中每只宠已经积累了哪些观测。

3. 继续优化胜率模型
   - 从“评分映射”升级到综合 HP、能量、存活数、观测充分度的模型。

### 优先级中

1. 引入更细的敌方技能候选权重
   - 不再只按简单启发式排序。
   - 可结合属性、能耗、已展示行为、当前局势。

2. 把更多事件接入暗箱推断
   - 敌方换宠
   - 能量变化
   - 状态施加/解除
   - 场地效果与印记

3. 对速度线加入更多上下文
   - 处理顺风、戏法空间、buff 层数等场景的更细化约束。

### 优先级低

1. 为明箱模式补一个独立入口页
   - 保留完整信息调试能力
   - 但不打扰暗箱主流程

2. 做更系统的文档拆分
   - 把历史功能报告留作归档
   - 把“当前状态”和“下一步企划”集中维护到少量文档中
